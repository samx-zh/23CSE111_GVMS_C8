# Problem Description
In general sense booking system has many problems when it comes to the manual way of booking and collecting data like trip fare, place, lack of tracking deliveries and trip status, poor coordination between customers, drivers and administrators, etc.
Now the best way to clear all these issues is by using digital way processing data that monitor and keep everything in order, that’s where GVBMS comes in handy.
Through we are aiming to solve majority of problems also create a good (admin-customer-driver) coordination when it comes to digital way every data will be stored and chance corruption and misrepresentation would be completely avoidable.

# How to run the code.

2. Open the project in any IDE (Eclipse/IntelliJ)
3. Compile all the classes
4. Run GoodsVehicleManagementSystem.java
